# Starlink Constellation Visualizer

Starlink Constellation Visualizer is a Meteor 3 + React application that ingests public Starlink orbital data, normalizes it into MongoDB, and renders live estimated satellite positions on a CesiumJS 3D globe using `satellite.js`.

The project is now containerized, so the recommended way to run it locally is with Docker.

## Features

- Ingests current Starlink orbital data from CelesTrak GP JSON feeds
- Falls back to TLE parsing when JSON is unavailable
- Normalizes orbital records into MongoDB with derived metadata
- Refreshes the catalog on startup and on a server schedule
- Continuously refreshes live propagated altitude and position samples
- Publishes filtered satellite subsets reactively with Meteor pub/sub
- Renders the active constellation on a Cesium globe
- Supports search, altitude filtering, favorites filtering, and display caps
- Shows a selected satellite's live details and short orbit path arc
- Includes a 24-hour forecast mode with playback and timeline scrubbing
- Supports coarse browser geolocation to show your approximate location
- Draws visibility lines for satellites currently above your local horizon

## Data Flow

1. Server startup creates indexes and ensures the app status document exists.
2. `refreshStarlinkCatalog` fetches Starlink data from CelesTrak.
3. The ingestion layer normalizes records and bulk-upserts them into MongoDB.
4. `refreshSatelliteLiveSamples` updates propagated live sample data on an interval.
5. The client subscribes to `satellites.filtered`, `satellites.filteredCount`, `satellites.single`, and `app.status`.
6. React reads Minimongo via `useTracker`.
7. The globe propagates display positions locally and updates the visualization in real time.

## Forecast And Location Features

- Forecast mode freezes a time anchor, lets you scrub up to 24 hours ahead, and can animate playback over the forecast horizon.
- The selected satellite gets a sampled orbit path overlay for easier trajectory inspection.
- Approximate location uses coarse browser geolocation rounded to a broad area instead of exact coordinates.
- When location is available, the globe highlights satellites currently above your local horizon.

## Cesium Setup

Cesium static assets are copied into `public/cesium` by `scripts/sync-cesium-assets.mjs` during `npm install`.

The client sets:

- `window.CESIUM_BASE_URL = "/cesium"`

and imports the Cesium widgets CSS directly.

## Running With Docker

### Prerequisites

- Docker Desktop or Docker Engine with Compose support

### Start The App

```bash
docker compose up --build
```

The app will be available at [http://localhost:3000](http://localhost:3000).

### Stop The App

```bash
docker compose down
```

### Optional Environment Variables

- `ORBIT_REFRESH_INTERVAL_MS`
  - server refresh cadence in milliseconds
  - default: `7200000` (2 hours)
  - values below the CelesTrak minimum are clamped on the server
- `ORBIT_FETCH_TIMEOUT_MS`
  - feed timeout in milliseconds
  - default: `30000`

Example:

```bash
ORBIT_FETCH_TIMEOUT_MS=45000 docker compose up --build
```

- `ghcr.io/<owner>/starlinktracker:latest`
- `ghcr.io/<owner>/starlinktracker:<sha>`
- branch and tag variants generated by the workflow

## Performance Notes

- The UI defaults to a server-side display cap of 250 satellites.
- The max visible control tops out at 1000.
- The globe updates positions on a fixed interval instead of every animation frame.
- The main publication sends only the current subset, not the entire catalog.

## Limitations

- Displayed positions are estimates derived from public orbital element sets, not direct telemetry.
- Server-side altitude filtering uses the latest ingested live sample, while the client globe keeps propagating positions locally.
- Favorites are local-only and stored in browser storage.
- Forecast positions are propagated estimates from the same public orbital model.
